Introduction

Where is the code maintained?

This is the code repository for the cataloging tool used in XERXES and Scripta Academica Lundensia. The problem addressed is how to share as much code between the services, and still be able to maintain the differences without too much trouble. The code store contains directories with code with various degree of specialization as follows:

The code repository of the cataloging tool
Directory Content
genericCommon code and files
authorCode used by authors only in both services
staffCode used by staff only in both services
scripta/genericCommon code and files in scripta
scripta/authorCode used by authors only in scripta
scripta/staffCode used by staff only in scripta
xerxes/genericCommon code and files in xerxes
xerxes/authorCode used by authors only in xerxes
xerxes/staffCode used by staff only in xerxes

The program ./bin/create_services.pl searches these directories and creates symbolic links to the content here in the directories used in the actual services. In searching it first looks in the generic directory to identify files, then it looks corresponding directories containing specialized code. If it finds such code it will use that, and if there is such code, it will take that instead.

The goal is to have almost everything except configuration files in the generic directory.
